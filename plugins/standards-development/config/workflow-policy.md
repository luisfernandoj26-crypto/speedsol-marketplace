# 🔄 Workflow Policy (DevOps Team)

## 🎯 OBJETIVO
Estandarizar flujo de trabajo del equipo de desarrollo.

---

## 🧑‍💻 GIT FLOW

- Usar branches por feature
- Main branch siempre estable
- Pull Requests obligatorios

---

## 🔍 CODE REVIEW

- Todo código debe ser revisado antes de merge
- Revisar:
  - arquitectura
  - seguridad
  - calidad
  - naming

---

## 🧪 TESTING

- Código nuevo debe incluir tests cuando aplique
- Validar regresiones antes de merge

---

## 🚨 REGLA CRÍTICA

No se permite merge sin aprobación y sin pasar validaciones del pipeline.
